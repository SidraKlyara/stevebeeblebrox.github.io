.mxn Files

Mixin files (.mxn) are used to tell Mixin what to do. Comments can be written on any new line. There is no character (Or set of characters) needed to define a comment. The only restriction on comments is that they can not start with a tag (see below) that would normally define a parameter. Comments can also be multiple lines. All parameters are defined in the following style: "nameOfTheParameter$value". Parameter names are written in camelCase. Accepted values varry between parameters (See below). Based off where Mixin files are located they are read different. After running Mixin once a folder called "mixin" will be created in the current Minecraft directory (Same location as the "mods" folder). Inside the "mixin" folder there are two sub folders: "mixins" and "exclusions".
When under "mixins", exactly 10 parameters must be defined. The "enabled" parameter accepts a lowercase boolean string that can act as a simple toggle on mixins. The "classTarget" parameter needs the full package path and name of the class Mixin should change (com.example.mod.SomeClassToChange). The "deleteOld" parameter takes a lowercase boolean string that tells Mixin if the target code should be deleted and replaced or used as a reference point to insert code before. The "deobfuscatedMethodTarget" uses the deobfuscated name of the method that is visible when coding with forge (getItem). The "obfuscatedMethodTarget" takes the method name when in a compiled environment like normal Minecraft (func_177251_a). To find method names use MCP Bot. "codeTarget" is what code Mixin should look for. What exactly this can be varies by what type of Node you are targeting. The "inject" parameter accepts the name of the method Mixin should call/insert (Mixin cannot accept an obfuscated and a nonobfuscated name to inject. It is up to mod authors to handle this themselves during production.). The "ownerClass" refers to where the above method is found. The package name is needed, but instead of periods the system's proper file separator should be used (com/stevebeeblebrox/mixin/asm/ASMHelper). "injectDescription" uses normal method descriptors (See #method-descriptors for more info on those) to explain to Java what the method that is being injected actually looks like. The "nodeType" defines what type of node is being targeted. Nodes are what .java classes become when compiled to .class files (See #supported-nodes for information on what types of nodes are supported).
When under "exclusions", 1 or more parameters can be defined. The only accepted parameter is "exclusion". The "exclusion" parameter takes a string containing a part of a package name (com.example.mod) or a class name (MainMod). Any class in the defined package or with the same name will not be edited by Mixin. Multiple exclusions can be defined in one file by using the "exclusion" tag on multiple lines.
.mxn files cannot be edited by normal text editors. Either use an IDE or Notepad++
